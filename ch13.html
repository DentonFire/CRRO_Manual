Chapter 13

Weekly Reporting

13.1 Weekly Task Report

Recipient: Fire Chief Kenneth Hedges

Due: Sunday evening (auto-generated via Power Automate)

Format: Microsoft Word (.docx) with DFD branding

Data Sources: SharePoint lists (Activities, Expenses, Events), Outlook Calendar

13.2 Report Structure

The weekly report follows a consistent structure that captures daily activities, events, budget expenses, and upcoming priorities:

Daily Activity Log

The main body of the report is a day-by-day table of activities:

Day Hours Activities
Monday 0730-2000 Monday Staff meeting. Event coordination for the week. [Description of key tasks and activities].
Tuesday 0800-2000 [Description of activities]. Candidate emails. [Additional tasks].
Wednesday OOS / hours N/A (or description of activities if working)
Thursday 0730-2030 [Description of activities]. CRR meeting. Candidate emails.
Friday 0730-2000 Candidate emails. Weekly report. [Additional tasks].
Saturday OOS / hours N/A (or description if working, e.g., Civil Service Exam day)

Events Section

List notable events attended or executed during the week (e.g., "Calhoun Middle School Recruiting", "Civil Service Exam").

Budget Expenses Section

List all P-Card expenditures for the week in the standard table format (Budget total, Amount, Account code, Description).

Next Week Preview

Brief day-by-day outline of planned activities for the following week. Example format:

  • M: Holiday / Staff meeting / [planned activity]
  • T: [planned activity]
  • W: [planned activity] / OOS
  • Th: CRR Meeting / [planned activity]
  • F: [planned activity]
  • S: OOS

13.3 Report Formatting

Weekly reports follow DFD branding standards:

  • DFD logo in header
  • Times New Roman 12pt font for body text
  • Red accent line (\#CC0000) under header
  • Clean, professional formatting

See the Power Automate Setup Guide (Appendix D) for automation workflow details.

13.4 Power Automate Integration

Weekly reports can be auto-generated from SharePoint lists using Microsoft Power Automate. The system pulls from:

  • CRR Weekly Activities list — Email logs, tasks completed
  • CRR Expenses list — Expense records with budget codes
  • ESO Events Log list — Community events attended
  • Outlook Calendar — Meetings and appointments

The report flow runs every Sunday at 8 PM CST, compiles the week's data, generates a formatted document, and sends a notification email. See the Power Automate Setup Guide for complete configuration details.
