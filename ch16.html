16.2 GitHub Pages — CRR Web Tools

All four CRR operational web tools are hosted on GitHub Pages, a free static hosting service integrated with the department’s GitHub organization account. This architecture was chosen for zero ongoing cost, high reliability, and simplicity of maintenance.

Repository Structure

Organization: github.com/dentonfire

Repository Live URL Key Files
Water-Shutoff dentonfire.github.io/Water-Shutoff/ index.html, water\_shutoff.html
Fire-Watch dentonfire.github.io/Fire-Watch/ index.html, PDF forms
Knox-Box dentonfire.github.io/Knox-Box/ index.html
After-the-Fire dentonfire.github.io/After-the-Fire/ index.html

Deployment Process

Changes to CRR web tools follow a simple deployment workflow:

  1. Edit the target file in the GitHub web interface (click pencil icon) or clone locally via git
  1. Commit changes with a descriptive commit message
  1. GitHub Pages automatically rebuilds and deploys within 1–2 minutes
  1. Hard-refresh the browser (Ctrl+Shift+R or Cmd+Shift+R) to verify changes

GitHub Pages Settings: Repository → Settings → Pages → Source: Deploy from branch → Branch: main, folder: / (root).

16.3 Firebase — Authentication & Database

The Water Shutoff Tracking System uses Google Firebase for user authentication and real-time data storage. Firebase provides a zero-cost backend at the department’s usage level.

Project Configuration

  • Firebase Console: console.firebase.google.com
  • Project Name: denton-fire-tools
  • Services Used: Authentication (Email/Password), Cloud Firestore (database)
  • Access Control: Only @cityofdenton.com email addresses can register
  • Email verification required before write access is granted

User Management

Users self-register with @cityofdenton.com email addresses. To manage users, navigate to Firebase Console → Authentication → Users tab. From this interface, administrators can view registered users, disable accounts, delete accounts, or force password resets. Users can also perform self-service password resets via the “Forgot Password” link on the login page.

Data Management

Firestore stores all shutoff records. Data can be exported via the Command Center’s “Download CSV” function with filters for status, type, and timeframe. A “Purge All” function exists for complete data deletion (password-protected). All data is transmitted over HTTPS and stored securely in Google’s cloud infrastructure.

16.4 SharePoint HUB

The DFD SharePoint HUB serves as the central internal platform for CRR resources, data lists, and team information. The CRR section is located at:

codenton.sharepoint.com/sites/DentonFire/SitePages/CRR\_Resources.aspx

SharePoint Lists for Reporting

Four SharePoint lists support the automated reporting system (see Chapter 13 for Power Automate integration):

  • CRR Weekly Activities — Email logs, tasks completed, activity type categorization
  • CRR Expenses — Expense records with budget codes and vendor information
  • ESO Events Log — Community events with attendance, type, and Hunter-attended flags
  • Weekly Reports — Generated report metadata with links to Word documents

HUB Content

The CRR Resources page provides department-wide access to all four CRR web tools (Water Shutoff Survey, Fire Watch Guidelines, Knox Box Guide, After the Fire), field guides, leadership overviews, and quick-reference materials. Operations and Prevention staff access these tools through bookmarked links on their mobile devices.

16.5 NEOGOV — Applicant Tracking

NEOGOV is the City of Denton’s applicant tracking system used for managing the firefighter hiring pipeline. The CRR Officer independently manages the fire department hiring workflow within NEOGOV, as fire department hiring operates under Texas Local Government Code Chapter 143 rather than standard City HR processes.

Key NEOGOV functions managed by CRR Officer:

  • Position Opportunity Announcement (POA) creation and publication
  • Application review and screening
  • Multi-stage pipeline management (PHS, polygraph, PAT, interviews, medical/psych)
  • Candidate communication and status updates
  • Disqualification processing and letter generation
  • Conditional and appointment letter generation

Coordinate with HR for system access, job posting visibility, and any cross-departmental requirements.

16.6 Constant Contact — Newsletters

Constant Contact is the email marketing platform used for both CRR and Recruiting newsletter distribution. The platform provides contact list management, email template design, scheduling, and analytics.

Account Details

  • Two primary newsletter series: CRR Newsletter (471 subscribers) and Ready to Serve Recruiting Newsletter (3,960 subscribers)
  • Analytics tracked: open rate, click-through rate, unsubscribe rate, bounce rate
  • Target open rate: 25%+
  • Newsletter archives accessible via conta.cc short links (documented in quarterly reports)

See Chapter 6 for detailed newsletter production workflows and editorial calendars.

16.7 Power Automate — Workflow Automation

Microsoft Power Automate (included with M365 licensing) automates the weekly and quarterly reporting pipeline. Five interconnected workflows capture data from email, ESO events, and expense notifications, then compile them into formatted reports.

Active Workflows

  1. Email Tracking — Creates tasks and logs from flagged incoming emails
  1. ESO Event Capture — Logs weekly community event attendance to SharePoint
  1. Expense Tracking — Captures Emburse notifications and assigns budget line items
  1. Weekly Report Generation — Compiles weekly data into formatted report (runs Sunday 8 PM CT)
  1. Quarterly Report Compilation — Aggregates 13 weekly reports into quarterly appraisal

Access Power Automate at make.powerautomate.com with @denton-tx.gov credentials. For complete setup instructions and troubleshooting, refer to the Power Automate Setup Guide (power\_automate\_setup.md).

16.8 Additional Systems

Emburse (P-Card Management)

Emburse provides electronic receipt capture and expense categorization for P-Card transactions. Expense notifications are automatically captured by the Power Automate expense tracking workflow. Manual reconciliation is performed periodically against department budget accounts.

Handshake (UNT Recruiting)

Handshake is the career services platform used at UNT for posting job opportunities and connecting with potential candidates, including the ELEVATE Athletes program. Account was finalized in Q1 FY 25-26.

Subtext (Text Messaging)

Subtext enables text message-based outreach to community members and recruiting contacts. Application setup was completed in Q1 FY 25-26. This platform extends CRR’s reach beyond email to reach audiences who prefer SMS communication.

DroneSense (UAS Operations)

DroneSense provides flight logging, reporting, and operational management for the joint Public Safety UAS/DFR program. Reports are generated from DroneSense for UAS activity tracking.

VerifEYE (Polygraph)

VerifEYE replaced traditional polygraph methods in FY 25-26, offering remote testing capability via mobile app, enhanced accuracy, and cost savings. The vendor manages the platform; CRR Officer coordinates candidate scheduling and results collection.
