Chapter 17

System Administration & Succession Planning

This chapter addresses the practical considerations for maintaining CRR technology systems and ensuring continuity of operations during personnel transitions. Because many of these systems were built and are solely administered by the CRR Officer, documenting access, procedures, and institutional knowledge is critical.

17.2 CRR Web Tools — Technical Maintenance

The CRR web tools require minimal ongoing maintenance due to their static hosting architecture. However, the following tasks should be performed on a regular basis:

Monthly Maintenance

  • Verify all four tool URLs load correctly and function as expected
  • Review Firebase Authentication for any unauthorized account registrations
  • Check Water Shutoff Command Center for data integrity
  • Test email routing (submit a test report and verify email generation)

Quarterly Maintenance

  • Export Water Shutoff data via CSV for quarterly reporting
  • Review and update unit dropdown options if apparatus assignments change
  • Check GitHub Pages deployment settings remain configured correctly
  • Review Firebase usage metrics to ensure within free-tier limits

Annual Maintenance

  • Review all tool content for accuracy (contact emails, procedures, links)
  • Verify Building Safety contact information in After the Fire guide (940-349-8600)
  • Update Fire Watch PDF forms if IFC requirements change
  • Review Knox Box guide for any IFC Section 506 updates
  • Confirm @cityofdenton.com email domain restriction is still appropriate

Common Troubleshooting

Users can’t log in: Verify email ends with @cityofdenton.com; check if email is verified in Firebase console; suggest “Forgot Password” reset. City email scanners may auto-click verification links causing “expired” messages — user can typically log in anyway.

Map not loading: Check internet connection; clear browser cache; try different browser. Leaflet.js with OpenStreetMap tiles is the mapping stack.

Changes not appearing after commit: Wait 1–2 minutes for GitHub Pages rebuild; hard-refresh browser; check GitHub Actions tab for build errors.

Data not syncing: Check Firebase console for error messages; verify Firestore security rules are published; check browser console (F12) for JavaScript errors.

17.3 Power Automate Maintenance

The automated reporting system requires periodic monitoring to ensure reliability:

  • Check flow run history weekly for any failed runs (make.powerautomate.com → My Flows → Run History)
  • Verify weekly report generation each Monday morning
  • Update SharePoint list column choices if budget accounts or event categories change
  • Renew any connections that expire (Outlook, SharePoint connectors may require re-authentication)
  • Test the quarterly compilation flow before each quarter-end deadline

Common issues include SharePoint column name mismatches (internal names use \_x0020\_ for spaces), expired OAuth connections, and date formatting errors. Refer to the Power Automate Setup Guide for detailed troubleshooting procedures.

17.4 Succession Planning

The CRR Officer position is the sole administrator for many critical systems. The following succession documentation ensures continuity:

Critical Documentation Package

A successor should receive the following materials within the first week:

  1. This Operations Manual (comprehensive reference for all programs)
  1. Water Shutoff Technical Administrator Guide (detailed system documentation)
  1. Water Shutoff Field Guide for Firefighters (end-user reference)
  1. Water Shutoff Leadership Overview (executive summary for chiefs)
  1. Power Automate Setup Guide (complete workflow documentation)
  1. Access credentials for all systems listed in Section 17.1
  1. Current quarterly program appraisal (most recent operational snapshot)
  1. Active candidate list and hiring pipeline status from NEOGOV

Knowledge Transfer Timeline

Recommended 2–4 week overlap period for outgoing and incoming CRR Officer:

Week Focus Area Key Activities
Week 1 Systems & Access Transfer all credentials, tour platforms, review this manual
Week 2 Recruiting Pipeline NEOGOV walkthrough, active candidates, vendor contacts (FPSI, VerifEYE, Concentra, Pepper)
Week 3 CRR Programs Newsletter production, event coordination, tool maintenance, community contacts
Week 4 Reporting & Admin Weekly report generation, budget reconciliation, accreditation files, Chief Hedges briefing

Programs Requiring Succession Planning

Several programs identified in Q1 FY 25-26 require succession planning beyond the CRR Officer position:

  • Clown Program — Currently inactive; requires identifying and training performers if program resumes
  • Explorer/Scout Program — BSA renewed in Q1; requires instructor with current BSA registration and safety training
  • DFR/UAS Program — Shared responsibility; requires FAA Part 107 certification and DroneSense access
  • Citizens Fire Academy — Coordination role; requires knowledge of curriculum, guest speakers, and logistics

17.6 Security Considerations

All CRR web tools enforce authentication via Firebase with @cityofdenton.com email restrictions. Data is encrypted in transit (HTTPS) and at rest (Google Cloud). No sensitive personal information (SSN, medical records) is stored in any CRR-managed system. Candidate personal information is managed within NEOGOV’s secure environment and City HR systems.

Key security practices to maintain:

  • Never store sensitive candidate data outside of NEOGOV or official City systems
  • Verify Firebase security rules remain published and enforced after any code changes
  • Regularly review registered Firebase users for unauthorized accounts
  • Ensure GitHub repository remains under the dentonfire organization (not personal accounts)
  • Update the Water Shutoff purge password periodically and limit knowledge of current password
  • Follow City of Denton IT security policies for all data handling and system access
